# Pertemuan 3 - REST API Design

## Tujuan

Mahasiswa mampu mendesain endpoint REST API.

## Materi

- Resource-oriented design
- URI naming convention
- API versioning
- Error response dan pagination

## Praktik

1. Desain resource untuk domain perpustakaan: books, members, loans.
2. Buat daftar endpoint dengan format `/api/v1/...`.
3. Tentukan contoh response sukses dan error.
4. Tentukan format pagination untuk endpoint list.

## Output

Dokumen desain endpoint REST API.
