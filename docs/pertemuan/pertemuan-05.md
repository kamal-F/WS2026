# Pertemuan 5 - Implementasi REST API CRUD

## Tujuan

REST API fungsional.

## Materi

- Controller dan routing
- Request/response typing
- Validation dasar

## Praktik

1. Coba `GET /api/v1/books`.
2. Tambah data dengan `POST /api/v1/books`.
3. Ambil detail dengan `GET /api/v1/books/:id`.
4. Ubah data dengan `PUT /api/v1/books/:id`.
5. Hapus data dengan `DELETE /api/v1/books/:id`.
6. Uji validasi dengan body request yang tidak lengkap.

## Output

REST API CRUD dengan validasi dasar.
