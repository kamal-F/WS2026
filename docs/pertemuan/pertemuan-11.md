# Pertemuan 11 - Pengantar Microservice Architecture

## Tujuan

Mahasiswa memahami desain microservice.

## Materi

- Monolith vs microservice
- Service boundary
- Data ownership

## Praktik

1. Ambil domain perpustakaan dari API sebelumnya.
2. Pisahkan kandidat service: catalog service, member service, loan service.
3. Tentukan data ownership setiap service.
4. Buat diagram komunikasi antar service.
5. Tandai operasi yang cocok memakai REST dan operasi yang cocok memakai event.

## Output

Desain awal microservice untuk studi kasus perpustakaan.
