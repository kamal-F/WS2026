# Pertemuan 15 - Integrasi Arsitektur Web Service Modern

## Tujuan

Mahasiswa mampu memilih arsitektur komunikasi layanan yang tepat.

## Materi

- REST + Message Queue
- REST + gRPC
- API Gateway konsep
- Observability dan logging konsep

## Praktik

1. Rancang arsitektur final untuk sistem perpustakaan.
2. Tentukan endpoint publik lewat API Gateway.
3. Tentukan komunikasi internal memakai REST, gRPC, atau message queue.
4. Tentukan event penting yang perlu dilog.
5. Buat alasan pemilihan arsitektur.

## Output

Dokumen analisis arsitektur web service modern.
