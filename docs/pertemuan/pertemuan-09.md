# Pertemuan 9 - API Testing & Documentation

## Tujuan

API terdokumentasi dan teruji.

## Materi

- Postman
- Swagger/OpenAPI
- Contract-based API documentation

## Praktik

1. Buat collection Postman untuk endpoint books.
2. Tambahkan test sederhana untuk status code `200`, `201`, `400`, dan `404`.
3. Tulis spesifikasi OpenAPI untuk endpoint books.
4. Cocokkan request/response aktual dengan dokumentasi.

## Output

API terdokumentasi dan memiliki skenario testing dasar.
